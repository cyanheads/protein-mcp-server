# 0.1.1 — 2026-06-18

## Added

- **`protein_get_structure` `pdbId`** — `best_available` picks that resolve to an experimental structure now carry the chosen PDB entry id (and its `title`) as first-class output fields, rendered as `**PDB:**` in `content[]`. `id` stays the queried UniProt accession; an agent no longer has to parse the coordinate URL to cite the structure. The title lookup is best-effort — a failed fetch never drops the structure. [#5](https://github.com/cyanheads/protein-mcp-server/issues/5)

## Changed

- **`protein_track_ligands` `structures_with_ligand`** — a valid component with zero containing structures now returns an empty `structures` set with `totalCount: 0` and an explanatory notice, instead of throwing NotFound. Mirrors `protein_search_structures`' empty-hits behavior. [#4](https://github.com/cyanheads/protein-mcp-server/issues/4)

## Fixed

- **RCSB zero-result searches** — a query matching nothing returns an empty result set sub-second instead of a `ServiceUnavailable` "endpoint may be temporarily unavailable" error after 4 retries. RCSB answers a zero-result query with `204 No Content`; `fetchJson` gains an `onEmptyBody` option that the search layer uses to resolve 204 / empty bodies to `{ total_count: 0, result_set: [], facets: [] }`. The empty-body guard still throws for non-search endpoints (AlphaFold, UniProt), where a blank body is a genuine transient failure. Affects `protein_search_structures`, `protein_analyze_collection`, and `protein_track_ligands` modes `find_ligand` and `structures_with_ligand`. [#4](https://github.com/cyanheads/protein-mcp-server/issues/4)
- **`protein_track_ligands` missing-param guards** — the three mode-specific required-input guards (`query` for `find_ligand`, `comp_id` for `structures_with_ligand`, `pdb_id` for `binding_site`) now throw a new `missing_param` reason with code `InvalidParams` (`-32602`) instead of NotFound (`-32001`), matching `protein_search_structures` and `protein_find_similar`. A client branching on error code sees a validation error for bad input rather than a misleading not-found. [#6](https://github.com/cyanheads/protein-mcp-server/issues/6)
