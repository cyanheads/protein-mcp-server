# 0.3.2 — 2026-07-03

## Added

- **`protein_track_ligands` `find_ligand`** — ligand objects carry a new `depositionCount` field (PDB entries containing the component); candidates are sorted by it, most-deposited first. ([#17](https://github.com/cyanheads/protein-mcp-server/issues/17))
- **`protein_find_similar` `by:"sequence"`** — hits carry a new optional `entityId` field with the raw matched polymer-entity ID (e.g. `1A00_1`). ([#18](https://github.com/cyanheads/protein-mcp-server/issues/18))

## Changed

- **`protein_track_ligands` `find_ligand`** — candidate ranking is deposition-frequency order instead of RCSB's name-string match order, so the canonical component (e.g. `HEM` for "heme") surfaces first instead of a near-namesake. ([#17](https://github.com/cyanheads/protein-mcp-server/issues/17))
- **`protein_find_similar` `by:"sequence"`** — `id` is now the bare, chainable PDB entry ID (e.g. `1A00`) instead of the raw polymer-entity ID (`1A00_1`); the entity ID moved to the new `entityId` field. ([#18](https://github.com/cyanheads/protein-mcp-server/issues/18))
- **`protein_track_ligands` `structures_with_ligand`** — entries drop the constant, meaningless `score` field in favor of `resolution` (Å) and sort best-resolution-first, instead of the undocumented PDB-ID-ascending order. ([#19](https://github.com/cyanheads/protein-mcp-server/issues/19))
