# 0.3.3 — 2026-07-03

## Added

- **`protein_search_structures` / `protein_analyze_collection`** — numeric histogram facet buckets (resolution, molecular_weight) carry new `rangeFrom`/`rangeTo` fields describing the bucket's half-open `[rangeFrom, rangeTo)` interval, computed from the facet's fixed bin width. Resolves ambiguity in bare boundary labels like `1.0`. ([#21](https://github.com/cyanheads/protein-mcp-server/issues/21))

## Changed

- **`protein_search_structures`** — description now documents that a facet on an actively-filtered dimension (e.g. the organism facet while `organism` is set) shows unfiltered sibling values by design, matching RCSB's standard drill-down behavior. ([#21](https://github.com/cyanheads/protein-mcp-server/issues/21))

## Fixed

- **`protein_analyze_collection`** — `interval`, `max_resolution`, and `bucket_limit` now coerce stringified numeric input (`"0.5"` → `0.5`), so clients that serialize tool arguments as strings can reach the numeric-histogram path. `interval: "year"/"month"/"quarter"` still resolve to the date-period arm. ([#15](https://github.com/cyanheads/protein-mcp-server/issues/15))
- **`protein_track_ligands` `find_ligand`** — the candidate pool is no longer capped by the display `limit` before re-ranking by deposition frequency, so a low-name-ranked canonical component (e.g. `HEM` for "heme") surfaces even at a small `limit`. ([#17](https://github.com/cyanheads/protein-mcp-server/issues/17))
