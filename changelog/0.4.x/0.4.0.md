# 0.4.0 — 2026-07-03

## Added

- **`protein_get_annotations`** — new `limit` input (1–200, default 50) caps features, natural variants, and InterPro domains independently, so a densely-annotated protein (a well-studied protein can carry 150+ natural variants) no longer returns an unbounded payload dominated by one class. A capped class is disclosed in the response `notice` ("Showing N of M \<class\>"), an empty requested class gets a "No \<class\> present" note, and `truncated: true` is set when any class was capped. ([#20](https://github.com/cyanheads/protein-mcp-server/issues/20))

## Changed

- **`protein_compare_structures`** — tool description and the `tmScore` output field now document that TM-score is length-normalized and can shift sharply between structures that differ only by a terminal residue or two, since the greedy superposition can settle into a worse local optimum; cross-check `rmsd` and `alignedResidues` to spot such cases. ([#22](https://github.com/cyanheads/protein-mcp-server/issues/22))
