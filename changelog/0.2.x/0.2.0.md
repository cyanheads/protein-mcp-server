# 0.2.0 — 2026-07-03

## Added

- **`protein_find_similar`** — new optional `ticket_id` input. When set on a `by:"structure"` call, polls the existing Foldseek job instead of resolving coordinates and submitting a new search; `pdb_id`/`uniprot`/`databases` are ignored. New error reason `ticket_not_found` when the ticket is invalid or expired. ([#8](https://github.com/cyanheads/protein-mcp-server/issues/8))
- **`protein_compare_structures`** — new optional `resume` input: an array of `{ a, b, uuid }` entries copied from a prior response's `pairs[]`. A pair whose labels match a `resume` entry (order- and case-insensitive) polls the given UUID instead of submitting a fresh alignment job; unmatched pairs still submit fresh. New error reason `resume_pair_unmatched` when a `resume` entry's labels don't match any pair generated from `structures` + `reference`. ([#8](https://github.com/cyanheads/protein-mcp-server/issues/8))
- **`AlignmentService.resumePair`** / **`FoldseekService.resume`** — service-level resume methods backing the two tools above. Both reuse the existing polling step and never throw; `resumePair` treats a `404` as `computing` (the RCSB alignment API returns the same 404 for an expired UUID and a job still in flight, so it can't safely report not-found), while `resume` distinguishes a Foldseek `400 "invalid ID"` as a genuine `not_found`.

## Changed

- **`protein_compare_structures`** — `structures` cap widened from a hardcoded `.max(10)` to `.max(25)`, matching the existing `PROTEIN_MAX_COMPARE_STRUCTURES` config ceiling (2–25, default 10). A request exceeding the configured cap now surfaces a truncation notice (previously silent). Tool and field descriptions reworded off the hardcoded "2–10" language to describe the configured cap. ([#11](https://github.com/cyanheads/protein-mcp-server/issues/11))
- **Dependencies:**
  - `@cyanheads/mcp-ts-core` ^0.10.9 → ^0.10.10
  - `@biomejs/biome` 2.5.1 → 2.5.2 (dev)
  - `@types/node` 26.0.0 → 26.1.0 (dev)

## Security

- **`js-yaml`** — the dependency refresh (`@cyanheads/mcp-ts-core` bump + lockfile re-resolve) moves the transitive `js-yaml` resolution from 3.14.2 to 3.15.0, clearing [GHSA-h67p-54hq-rp68](https://github.com/advisories/GHSA-h67p-54hq-rp68) (quadratic-complexity ReDoS in `js-yaml < 3.15.0`). `bun audit`: 1 moderate → 0.
